This guide walks through two complete workflow examples so you can start using the AzuraCast node right away.

## Example 1: Get Now Playing data from your radio station

The Now Playing endpoint is public — no API key required. This is the fastest way to verify that your credentials and node setup are working.

<Steps>
  <Step title="Add the AzuraCast node">
    Open a workflow in n8n, click **+** to add a node, search for `AzuraCast`, and select the **AzuraCast** node.
  </Step>
  <Step title="Connect your credentials">
    In the node settings, click the **Credentials** dropdown and select an existing **AzuraCast API** credential, or create a new one.

    For this example you only need the **Base URL** — leave the API Key empty.

    See [Credentials](/credentials) for setup instructions.
  </Step>
  <Step title="Select the Resource">
    In the **Resource** dropdown, select **Public Now Playing**.
  </Step>
  <Step title="Select the Operation">
    In the **Operation** dropdown you have two options:

    - **Get now playing** — returns Now Playing data for all stations at once as an array.
    - **Get now playing for station** — returns Now Playing data for a single station. When selected, a **Station ID** field appears; enter the short name or numeric ID of your station (e.g. `azuratest_radio` or `1`).

    For a quick test, select **Get now playing** to fetch all stations.
  </Step>
  <Step title="Execute and inspect the response">
    Click **Execute node**. The output will contain one item per station, each with fields such as:

    ```json
    {
      "station": {
        "id": 1,
        "name": "AzuraTest Radio",
        "shortcode": "azuratest_radio",
        "listen_url": "https://radio.example.com/radio/8000/radio.mp3"
      },
      "now_playing": {
        "song": {
          "title": "Example Song",
          "artist": "Example Artist",
          "album": "Example Album"
        },
        "elapsed": 42,
        "remaining": 178
      },
      "listeners": {
        "current": 12,
        "total": 18
      },
      "live": {
        "is_live": false
      }
    }
    ```

    You can reference any of these fields in downstream nodes using n8n expressions such as `{{ $json.now_playing.song.title }}`.
  </Step>
</Steps>

<Tip>
  Connect the AzuraCast node to a **Set** node or **Item Lists** node to extract only the fields you need before passing data to other services.
</Tip>

## Example 2: Upload media to a station

This example shows how to use a multipart file upload operation. An API key is required.

<Steps>
  <Step title="Provide a binary file to the workflow">
    Before the AzuraCast node, add a node that produces a binary item — for example:
    - A **Read Binary File** node pointing to a local audio file
    - An **HTTP Request** node that downloads an audio file
    - A **Webhook** node that receives a file upload

    The binary data should be stored in a named property (the default is `data`).
  </Step>
  <Step title="Add the AzuraCast node">
    Click **+**, search for `AzuraCast`, and select the **AzuraCast** node.
  </Step>
  <Step title="Connect credentials with an API key">
    Select or create an **AzuraCast API** credential that includes a valid **API Key**. Media upload is a protected endpoint.
  </Step>
  <Step title="Select the Resource">
    In the **Resource** dropdown, select **Stations Media**.
  </Step>
  <Step title="Select the Operation">
    In the **Operation** dropdown, select **Upload media**.

    The following fields will appear:

    | Field | Description |
    |-------|-------------|
    | **Station ID** | The numeric ID or shortcode of the target station, e.g. `1` or `azuratest_radio` |
    | **Input Data Field Name** | The name of the binary property in the incoming item that holds the file (default: `data`) |
    | **File Field Name** | The multipart form-data field name to use when sending the file to the API (default: `file`) |
  </Step>
  <Step title="Fill in the required fields">
    - Set **Station ID** to your station's ID.
    - Set **Input Data Field Name** to the binary property name from the upstream node (e.g. `data`).
    - Leave **File Field Name** as `file` unless your AzuraCast version requires a different field name.
  </Step>
  <Step title="Execute and verify">
    Click **Execute node**. A successful upload returns a JSON response with the uploaded file's metadata:

    ```json
    {
      "id": 42,
      "song_id": "abc123def456",
      "text": "Example Artist - Example Song",
      "title": "Example Song",
      "artist": "Example Artist",
      "album": "",
      "path": "example-song.mp3",
      "length": 220,
      "length_text": "3:40"
    }
    ```

    The uploaded file will be visible in the AzuraCast Media Manager for the station you specified.
  </Step>
</Steps>

<Note>
  The AzuraCast node supports all 263 operations from the official AzuraCast OpenAPI specification. Resources are grouped by API tag — for example, **Administration General**, **Administration Users**, **Stations Playlists**, **Stations Web Hooks**, and more. Explore the **Resource** dropdown to see all available groups.
</Note>
