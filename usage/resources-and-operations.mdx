Every AzuraCast node interaction starts by selecting a **Resource** and an **Operation**. These two dropdowns determine which API endpoint the node calls and which fields are displayed.

## Resource

A resource maps to an official AzuraCast API tag — a logical grouping of related endpoints. For example, the `Stations Media` resource contains all endpoints for managing media files on a station.

Resource values are derived from the OpenAPI tag names. The internal value follows the pattern `resource_<tag_slug>` (for example, `resource_stations_media`).

## Operation

An operation maps to a specific API endpoint within the selected resource. Operation labels are generated from the OpenAPI `operationId` field: underscores and hyphens are replaced with spaces, camelCase is split, and the result is title-cased.

For example, `getStationMedia` becomes **Get Station Media**, or when scoped to the Stations Media resource and redundant tokens are removed, it may appear as **Get** or **List**.

The operation's value is the raw `operationId` string from the OpenAPI spec.

## How operation labels are generated

Operation display names go through several normalization steps:

1. The `operationId` string (for example, `api_admin_backups_list`) is split on underscores and hyphens, and camelCase boundaries are detected.
2. Known acronyms (`API`, `DJ`, `HLS`, `HTTP`, `SFTP`, `URL`, `XML`, `ID`) are uppercased.
3. Tokens from the resource name that appear in the operation name are removed to avoid redundancy (for example, **Get Station Media** becomes **Get** when displayed under the Stations Media resource).
4. If two operations in the same resource produce the same label, the HTTP method is appended in parentheses (for example, **Update (PUT)**).

## Legacy operationId field

Workflows created before the unified node can still pass an `operationId` parameter directly. The execution runtime resolves the operation from the global operation map using that value. The `operationId` takes precedence over the **Resource** + **Operation** selector pair when it is set and non-empty.

<Note>
The `operationId` field is not shown in the default node UI. It is only used when present in saved workflow JSON from earlier node versions. New workflows should use the **Resource** and **Operation** dropdowns.
</Note>
