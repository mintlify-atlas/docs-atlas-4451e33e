## Prerequisites

You need Node.js (v22 recommended) and npm installed before getting started.

## Setting up the dev environment

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/renebell0/n8n-nodes-azuracast.git
    cd n8n-nodes-azuracast
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm ci
    ```
  </Step>

  <Step title="Build the project">
    ```bash
    npm run build
    ```

    This compiles TypeScript and produces the `dist/` output used by n8n.
  </Step>

  <Step title="Verify operation coverage">
    ```bash
    npm run verify:operations
    ```

    This fetches the live AzuraCast OpenAPI source and confirms that the snapshot covers all 263/263 operation IDs.
  </Step>
</Steps>

## Development scripts

| Script | Description |
|---|---|
| `npm run build` | Compile TypeScript to `dist/` |
| `npm run dev` | Run n8n node dev mode |
| `npm run lint` | Lint source files |
| `npm run lint:fix` | Lint and auto-fix |
| `npm run simulate:local` | Run local simulation against generated operations |
| `npm run verify:operations` | Verify 263/263 OpenAPI operation coverage |
| `npm run verify:ui` | Verify UI metadata rendering integrity |
| `npm run validate:publish` | Full pre-publish check (lint + build + verify + simulate + pack) |

### Running the linter

```bash
npm run lint
```

To auto-fix lint issues:

```bash
npm run lint:fix
```

### Running local simulation

The simulation script exercises the generated operations locally to verify request/response handling across JSON, binary, multipart, redirect, and auth scenarios:

```bash
npm run simulate:local
```

Run this before publishing to catch any runtime issues that the TypeScript compiler would not catch.

## Verification scripts

Two verification scripts run as part of `validate:publish` and can be run individually during development.

**`verify:operations`** fetches the official AzuraCast OpenAPI source and checks that:
- All operation IDs in the live spec are present in the snapshot
- No extra or missing operations exist in the snapshot
- Operation metadata (method, path, tag, public flag) has not drifted
- The domain manifest (`azuracast.domains.json`) covers all operations
- Generated module files in `properties/` and `execute/` match the domain count
- `package.json` `n8n.nodes` lists exactly the expected built node path

**`verify:ui`** loads the compiled `properties/index.js` and checks that:
- Resource options are present and have a valid default
- Each resource has exactly one operation selector
- Every operation option has a non-empty `name`, `value`, and `action`
- No operation label looks like a path (e.g. starts with `/` or contains `[`, `{`)
- No operation label is longer than 8 words
- No duplicate operation labels or values exist within or across resources

<Note>
`verify:ui` requires a successful `npm run build` first, because it imports from `dist/`.
</Note>

## Code structure

The repository is organized as follows:

```
n8n-nodes-azuracast/
├── credentials/
│   └── AzuraCastApi.credentials.ts   # AzuraCast API credential type
├── nodes/
│   └── AzuraCast/
│       ├── AzuraCast.node.ts          # Main node entry point
│       ├── AzuraCast.shared.ts        # Shared helpers and execution runtime
│       ├── azuracast.openapi.snapshot.json  # Generated OpenAPI snapshot
│       ├── azuracast.domains.json     # Generated domain groupings manifest
│       ├── properties/                # Per-domain UI property definitions
│       │   ├── index.ts
│       │   ├── resources.ts
│       │   ├── <domain>.operations.ts
│       │   └── <domain>.fields.ts
│       └── execute/                   # Per-domain execution handlers
│           ├── index.ts
│           └── <domain>.ts
├── scripts/
│   ├── generate-operations.mjs        # Fetches OpenAPI and generates snapshot
│   ├── generate-domain-nodes.mjs      # Generates properties/ and execute/ from snapshot
│   ├── verify-operations.mjs          # Verifies operation coverage against live OpenAPI
│   ├── verify-ui-metadata.mjs         # Verifies UI metadata rendering integrity
│   └── simulate-local.mjs             # Local simulation for request/response behavior
└── dist/                              # Compiled output (not committed)
```

### `nodes/AzuraCast/`

The main node. `AzuraCast.node.ts` is the n8n node entry point. It reads `resource` and `operation` parameters, looks up the matching executor from `execute/index.ts`, and calls it. `AzuraCast.shared.ts` contains the shared execution runtime that handles parameter resolution, credential injection, request dispatch, and response formatting.

### `nodes/AzuraCast/properties/`

Per-domain property definitions, generated by `generate-domain-nodes.mjs`. Each domain produces two files:

- `<domain>.operations.ts` — the operation selector for that domain/resource
- `<domain>.fields.ts` — the per-operation field properties (path, query, and body parameters)

These are collected and exported by `properties/index.ts` as `azuraCastNodeProperties`.

### `nodes/AzuraCast/execute/`

Per-domain execution handlers, also generated by `generate-domain-nodes.mjs`. Each domain gets a `<domain>.ts` file that maps operation IDs to executor functions. All executors call `executeOperationById` from `AzuraCast.shared.ts`, which resolves the full operation metadata from the snapshot and dispatches the request.

### `credentials/`

Contains `AzuraCastApi.credentials.ts`, the n8n credential type definition. It declares the `baseUrl` and `apiKey` fields and implements credential testing using a public and an authenticated AzuraCast endpoint.

### `scripts/`

Code generation and verification tools. See [Generating operations](/development/generating-operations) for details on the generation flow.
