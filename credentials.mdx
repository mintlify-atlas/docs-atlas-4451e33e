title: Credentials

The AzuraCast node uses the **AzuraCast API** credential type. You need to create one before any AzuraCast node can make requests.

## Credential fields

| Field | Required | Description |
|-------|----------|-------------|
| **Base URL** | Yes | The root URL of your AzuraCast server, e.g. `https://radio.example.com` |
| **API Key** | No\* | Your AzuraCast API key. Optional for public endpoints; required for protected endpoints. |

\* An API key is not required to call public endpoints such as Now Playing. It is required for any endpoint that needs authentication.

## Create AzuraCast API credentials

<Steps>
  <Step title="Open the Credentials panel">
    In n8n, click **Credentials** in the left sidebar, then click **Add credential**.
  </Step>
  <Step title="Search for AzuraCast API">
    Type `AzuraCast` in the search box and select **AzuraCast API** from the list.
  </Step>
  <Step title="Enter your Base URL">
    In the **Base URL** field, enter the root URL of your AzuraCast installation — for example, `https://radio.example.com`. Do not include a trailing slash or the `/api` path segment; the node resolves the API base path automatically.
  </Step>
  <Step title="Enter your API Key (optional)">
    If you need to call protected endpoints, paste your AzuraCast API key into the **API Key** field. Leave it empty if you only need public endpoints.
  </Step>
  <Step title="Test and save">
    Click **Test connection** to verify the credential, then click **Save**.
  </Step>
</Steps>

## How authentication works

When an API key is provided, the node adds two headers to every request:

```
X-API-Key: <your-api-key>
Authorization: Bearer <your-api-key>
```

When no API key is set, requests are sent without authentication headers. Public AzuraCast endpoints accept unauthenticated requests.

## Credential test behavior

The **Test connection** button sends a lightweight GET request to confirm the credential is valid:

- **Without API key** — tests `GET /api/nowplaying` (a public endpoint that requires no authentication).
- **With API key** — tests `GET /api/frontend/account/me` (an authenticated endpoint that returns the account details for the key owner).

<Note>
  A successful test without an API key only confirms that your Base URL is reachable. It does not verify that the API key is correct. If you set an API key, the test also confirms that the key is valid and authorized.
</Note>

## Finding your AzuraCast API key

<Steps>
  <Step title="Log in to AzuraCast">
    Open your AzuraCast station dashboard and log in with an administrator or station manager account.
  </Step>
  <Step title="Go to your profile">
    Click your username or avatar in the top-right corner, then select **My Account** (or **Profile**).
  </Step>
  <Step title="Generate or copy your API key">
    Scroll to the **API Keys** section. Create a new key or copy an existing one.
  </Step>
</Steps>

<Warning>
  Treat your API key like a password. Anyone with this key can make API calls on your behalf, including protected administrative operations.
</Warning>

## Public vs protected endpoints

AzuraCast has two categories of endpoints:

- **Public endpoints** — accessible without authentication. These include Now Playing data, station listings, and public schedule information. You can use these with an empty API Key field.
- **Protected endpoints** — require a valid API key. These include station management, media uploads, playlist edits, user administration, and any write operations.

If a node operation returns a `401 Unauthorized` or `403 Forbidden` error, add an API key to your credential and re-test.

For a full reference of which endpoints are public or protected, see the [AzuraCast API documentation](https://www.azuracast.com/docs/developers/apis/).
